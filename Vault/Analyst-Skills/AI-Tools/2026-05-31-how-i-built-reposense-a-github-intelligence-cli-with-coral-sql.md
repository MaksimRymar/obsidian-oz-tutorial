## What’s new and why it matters
Every developer I know has the same problem: 300 open issues, 40 stale PRs, a security label buried somewhere in the noise — and no fast answer to what actually needs attention right now? I built RepoSense to answer that question in under 10 seconds, for any public GitHub repo, with one terminal command and no dashboard. The Idea RepoSense is a terminal intelligence layer for GitHub repos. You point it at any repo and get: The oldest unattended issues (triage) Non-draft PRs that have been waiting too long (stale-prs) What shipped in the last two weeks (release-notes) What Hacker News is saying…
