## What’s new and why it matters
mtproto proxies are annoying -- half the lists out there have endpoints that died months ago. i just wanted something that actually pulls live ones from telegram channels and only keeps the ones that work with fake-tls. so i patched together a scraper that picks proxies from public channels, tests every single one for connectivity and fake-tls support, then publishes the survivors. dead ones get dropped on each run, so the list stays current. there's also a live json endpoint if you want to consume it programmatically. if you're tired of copy-pasting dead proxies: https://github.com/Yagami200/…
