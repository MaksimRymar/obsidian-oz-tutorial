## What’s new and why it matters
Most beginners start Python by writing scripts. That is a good thing. A script is often the first place where Python feels useful. You write one file, run it, solve a small problem, and get the output. Maybe the script reads a CSV file. Maybe it renames files. Maybe it calls an API. Maybe it cleans some data. Maybe it generates a small report. For small tasks, this is completely fine. The problem starts when the same script slowly becomes responsible for too many things. One more condition is added. Then one more file input. Then one more API call. Then one error case. Then one database call.…
