## What’s new and why it matters
Code review has a new lazy question: "Did AI write this?" Sometimes that question matters. If private code was pasted into an unapproved tool, it matters. If generated code was shipped without understanding, it matters. If the team has a compliance policy, it matters. But as a review standard, that question is weak. The better question is: "Is this code correct, tested, maintainable, and owned by the person submitting it?" That standard catches bad AI code. It also catches bad human code.
