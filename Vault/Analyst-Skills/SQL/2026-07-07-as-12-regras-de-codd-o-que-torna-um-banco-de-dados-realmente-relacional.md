## What’s new and why it matters
As 12 Regras de Codd: o que torna um banco de dados realmente relacional? Quando falamos em bancos de dados relacionais, normalmente pensamos em tabelas, linhas, colunas, chaves primárias, SQL e consultas usando SELECT. Porém, para Edgar Frank Codd, o criador do modelo relacional, isso não era suficiente. Codd não queria apenas que os bancos de dados parecessem relacionais. Ele queria estabelecer critérios técnicos para dizer se um Sistema Gerenciador de Banco de Dados, ou SGBD, era de fato relacional.
