## What’s new and why it matters
Here's something most SQL tutorials skip entirely: SQL wasn't just a new syntax. It was a new way of thinking . Before SQL existed, most systems were built around procedural programming. You didn't describe a result — you described every step to get there. Fetch the data. Check a condition. Process row by row. Loop again. Handle the edge cases. Store the result. Repeat. It worked. But it was exhausting. And fragile. And hard to scale. Then SQL introduced an idea that sounds almost too simple: Tell the system what you want — not how to do it.
